## Source-backed Forensic Crawler vision

The exports consistently propose:

- a rights-aware crawler for images, video, articles, and structured data;
- a curated source registry;
- author, publisher, time, location, rights, license, and URL metadata;
- cryptographic hashing and immutable provenance;
- a structured event database, synchronized timeline, witness/source comparison, and contradiction detection;
- later photogrammetry, multi-view geometry, depth estimation, camera and movement analysis;
- visible separation of verified and interpolated reconstruction elements;
- later mobile capture using camera, LiDAR, IMU, and microphones; and
- later cinematic, free-camera, VR/AR, and forensic-package outputs.

These are historical WORKING requirements, not proof that the features are feasible, lawful, differentiated, validated, or complete.

## Historical proposals requiring correction or validation

| Historical proposal | Current disposition |
|---|---|
| “Trusted sources only” | A registry prioritizes and constrains discovery but never assigns truth. Every record is evaluated individually. |
| “Fair use candidate - archive copy” | Rejected as an automatic action. Default is metadata, locator, and restrained excerpt pending documented review. |
| Green/yellow/red confidence | Replaced by text state, icon, pattern, accessible color, rationale, and bounded uncertainty. |
| “Legal-grade” or court-admissible export | Prohibited until validated procedures, qualified legal review, reproducibility evidence, and independent assessment exist. |
| Blockchain-style provenance | Replaced by an append-only hash-chained audit log unless a concrete distributed-trust requirement later justifies more. |
| Broad social-media scraping | Out of scope. Any future source requires separate lawful-access, terms, privacy, authenticity, and abuse review. |
| Full 3D/VR/mobile platform as initial build | Deferred. Phase 1 is a bounded provenance, claim, timeline, contradiction, rights, and export vertical slice. |
| Automatic rights classification | Reframed as explainable decision support with mandatory human review for ambiguous or consequential cases. |

## New recommendations introduced by this implementation

- permanent opaque IDs for every authoritative record;
- deterministic, versioned JSON packages before database lock-in;
- separate source facts, observations, claims, inferences, annotations, and reconstruction elements;
- metadata-only Apollo 11 landing-time pilot using official NASA records and no copied raw PDF;
- separately authorized Charlie Kirk public-record fixture using exact locators, restrained paraphrases, procedural labels, and project-authored schematic geometry with no copied source pixels;
- separately authorized Renee Good public-record fixture using exact locators, restrained paraphrases, uncharged/open-investigation labels, response chronology, and project-authored qualitative street geometry with no copied source pixels;
- provider-neutral React/Vite local workspace, with future PostgreSQL/PostGIS and isolated Python workers;
- deny-by-default network architecture, even though no crawler network code is enabled in this phase; and
- automated schema, reference, rights, provenance, export, and interface checks.
